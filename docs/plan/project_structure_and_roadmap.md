# 프로젝트 구조 제안 및 구현 로드맵

## 구조 설계 원칙

> **"같은 엔진, 같은 사용자"면 합치고, "다른 인프라 또는 다른 사용자"면 분리**

---

## 프로젝트 A — ForFun Visual Studio (메인 프로젝트)

**1 + 2 + 4를 하나로 묶습니다.**

이 세 방향은 결국 같은 것을 만듭니다: **파티클 엔진 + 커스터마이징 UI + 캔버스**. 차이는 "결과물을 어떻게 쓰느냐"뿐입니다.

| 방향 | 역할 | 프로젝트 내 위치 |
|---|---|---|
| 1. 비주얼 아트 플랫폼 | 핵심 엔진 | 코어 모듈 |
| 2. 분위기 연출 도구 | 실용적 활용 모드 | "Ambient Mode" 탭/섹션 |
| 4. 디지털 아트 도구 | 창작 + 내보내기 | "Create Mode" 탭/섹션 |

### 왜 합치는가

- 세 방향 모두 **동일한 렌더링 엔진**(파티클 시스템, 물리 시뮬레이션, 레이어)을 공유합니다. 분리하면 같은 엔진을 세 번 만드는 셈입니다.
- UI 차이는 "모드 전환"으로 해결됩니다. Ambient Mode에서는 날씨 API 연동과 음악 비주얼라이저가 켜지고, Create Mode에서는 고해상도 내보내기와 세밀한 파라미터 에디터가 열리는 식입니다.
- 사용자 입장에서도 자연스럽습니다 — 분위기 연출하다가 마음에 드는 장면이 나오면 바로 이미지로 저장하는 흐름.

### 기술 스택

프론트엔드 중심. Canvas/WebGL, Web Audio API, 외부 API 연동 정도. 백엔드 없이도 대부분 동작 가능.

---

## 프로젝트 B — ForFun Code Lab (별도 프로젝트)

**3. 크리에이티브 코딩 교육 플랫폼을 독립시킵니다.**

### 왜 분리하는가

이건 근본적으로 **대상이 다릅니다**. 프로젝트 A는 "만드는 사람"을 위한 것이고, 프로젝트 B는 "배우는 사람"을 위한 것입니다.

- UI 패러다임이 완전히 다릅니다. A는 `컨트롤 패널 + 캔버스`이지만, B는 `코드 에디터 + 미리보기 + 학습 가이드`입니다. 이걸 한 프로젝트에 넣으면 둘 다 어중간해집니다.
- 콘텐츠 구조가 필요합니다 — 챌린지, 단계별 커리큘럼, 힌트 시스템 등. 이건 아트 도구와 공존하기 어렵습니다.
- 다만 프로젝트 A의 엔진을 **라이브러리로 추출**하여 B에서 import하면, 학습자가 실제 동작하는 엔진 위에서 코드를 수정하며 배울 수 있습니다.

### 프로젝트 A와의 관계

A의 코어 엔진을 npm 패키지나 공유 라이브러리로 분리하고, B가 이를 의존성으로 사용. 단방향 의존.

---

## 프로젝트 C — ForFun Together (별도 프로젝트)

**5. 소셜 인터랙티브 경험을 독립시킵니다.**

### 왜 분리하는가

인프라가 근본적으로 다릅니다.

- 프로젝트 A는 **정적 프론트엔드**로 충분합니다. GitHub Pages에 올리면 끝.
- 프로젝트 C는 **실시간 백엔드**가 필수입니다 — WebSocket 서버, 사용자 세션, 데이터 저장, 동시접속 관리. 서버 운영 비용과 확장성 고민이 생깁니다.
- 보안 이슈도 다릅니다. 사용자 업로드 이미지 관리, 부적절한 콘텐츠 필터링 등 A에는 없는 문제가 발생합니다.

### 가벼운 공유 기능은 A에 포함 가능

파라미터를 URL 쿼리스트링으로 인코딩하여 "이 장면을 공유" 링크를 만드는 정도는 백엔드 없이 가능합니다. 진짜 실시간 멀티플레이어, 방명록, 누적 캔버스 등은 C로 분리.

---

## 구현 순서

```
Phase 1:  프로젝트 A 코어 엔진 (현재 프로젝트를 리팩토링)
          └─ Canvas/WebGL 전환, 파티클 시스템 설계

Phase 2:  프로젝트 A 모드 확장
          ├─ Ambient Mode (날씨 API, 음악 연동)
          └─ Create Mode (내보내기, 레이어)

Phase 3:  코어 엔진을 공유 라이브러리로 추출

Phase 4:  프로젝트 B (교육 플랫폼, 공유 엔진 활용)

Phase 5:  프로젝트 C (소셜 기능, 백엔드 구축)
```

A를 먼저 탄탄하게 만들고 엔진을 추출하는 게 핵심입니다. B와 C는 A의 엔진 위에서 자연스럽게 파생됩니다.
